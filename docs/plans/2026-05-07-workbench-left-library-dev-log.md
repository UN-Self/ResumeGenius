# 工作台左侧栏与素材库优化开发日志

## 背景

工作台左侧素材区原本只承担文件上传和解析内容预览，随着后续要加入图案组件库、简历模板库、AI 调用能力，单一素材列表已经不适合继续扩展。

本轮调整目标：

- 将左侧栏从单一素材列表升级为接近 VS Code Activity Bar 的多功能入口。
- 完善文件/文件夹层级、上传反馈、导出反馈和图片预览。
- 为组件库、模板库预留左侧入口。

## 已完成改动

### 1. 素材文件树

- 增加根目录与递归文件夹层级。
- 文件夹支持子文件夹，最大 7 层。
- 文件夹支持三点菜单：重命名、删除。
- 删除文件夹会级联删除子文件夹与文件。
- 文件名和文件夹名禁止换行，长名称使用省略号。
- 根目录和文件夹支持展开/收起。

### 2. 上传能力

- 普通上传支持选择目标文件夹。
- 增加上传文件夹能力：
  - 上传后创建同名根文件夹。
  - 递归创建内部目录。
  - 只接受 PDF、DOCX、PNG、JPG、JPEG。
  - 不支持的文件忽略。
  - 超过 7 层的文件忽略。
- 上传文件夹增加进度弹窗，显示当前文件和已处理数量。

### 3. 解析内容工作区

- 左侧不再展示解析文本预览框。
- 点击文件后在中间打开 tab。
- 文本素材用类代码编辑器行号形式展示。
- 图片素材在中间直接预览。
- 关闭有未保存改动的素材 tab 时使用自定义保存确认弹窗。

### 4. 图片预览修复

- 后端增加图片文件读取端点 `GET /api/v1/assets/{asset_id}/file`。
- 上传文件保存到持久化目录 `./data/uploads:/app/uploads`。
- 修复同 hash 图片软删除后重传时只恢复数据库、不恢复文件的问题。

### 5. 导出 PDF 反馈

- 导出 PDF 增加进度弹窗。
- 进度来自后端异步任务 `progress` 字段。
- 用户能看到排队、渲染、下载准备等阶段。

### 6. 左侧 Activity Bar

- 左侧栏新增功能入口：
  - 文件
  - 图案组件库
  - 简历模板库
- 文件页保留现有素材树。
- 图案组件库和简历模板库先搭建空状态。

## 验证

- `npm run build` 通过。
- `go test ./internal/modules/intake` 通过。
- 本地 workbench 可通过 `http://localhost:3000/app/` 访问。
