# 简历 AI 编辑产品 PRD v2

日期：2026-04-23

## 1. 产品概述

### 1.1 产品名称

**ResumeGenius**

### 1.2 产品定位

一款面向简历优化场景的 AI 编辑产品。用户上传现有简历文件，系统解析后 AI 生成初版简历，用户可通过 AI 对话或 TipTap 所见即所得编辑器直接修改，最终通过服务端渲染导出 PDF。

### 1.3 核心设计理念

**HTML 是唯一数据源**：

- 不维护中间 JSON 结构，不维护 Patch 协议
- 用户像用 Word 一样直接编辑简历 HTML
- AI 对话修改直接返回 HTML
- 导出走服务端（chromedp），用于商业化权限控制
- 部署友好：2C2G 服务器，Docker Compose 一键部署

## 2. 功能边界

### 2.1 本版本做什么

- 上传简历文件（PDF / DOCX）并解析提取文本
- AI 根据提取的文本生成初始简历 HTML
- TipTap 所见即所得编辑器直接编辑简历
- AI 对话式修改（SSE 流式，AI 返回修改后的完整 HTML）
- 版本管理（HTML 快照）
- 服务端 PDF 导出（chromedp，付费控制）
- Git 仓库接入（提取 README、技术栈等信息）

### 2.2 本版本不做什么

- 不做投递系统
- 不做草图上传
- 不做移动端适配
- 不让 AI 直接控制排版数值（AI 通过 HTML/CSS 间接控制）
- 暂不做图片 OCR（后续通过云端 API 兜底）
- 暂不做多用户认证（预留接口）

## 3. 目标用户

### 3.1 主用户

- 已有简历，想快速改版式/风格/结构的求职者
- 对简历样式敏感、希望反复微调的用户
- 有明确修改目标，但不想自己从零搭建的用户

### 3.2 用户核心诉求

- 我已经有简历了，想快速重构并优化
- 我需要自由编辑，同时让 AI 帮我优化内容
- 我希望导出的 PDF 稳定、所见即所得
- 我希望 AI 给建议，但最后我自己决定

## 4. 核心功能

### 4.1 项目管理

- 项目 CRUD
- 文件上传（PDF / DOCX / PNG / JPG）
- Git 仓库 URL 接入
- 补充文本资料录入

### 4.2 文件解析与 AI 初稿

- PDF 解析（ledongthuc/pdf 提取文本块和内嵌图片）
- DOCX 解析（nguyenthenguyen/docx 提取段落、表格、样式）
- 将提取的文本发送给 AI
- AI 根据简历 HTML 模板骨架 + 用户资料生成完整简历 HTML
- 将 HTML 存入数据库作为初始草稿

### 4.3 AI 对话助手

- 多轮对话（SSE 流式响应）
- AI 读取当前简历 HTML 作为上下文
- 用户描述需求，AI 流式返回修改后的完整 HTML
- 用户点击"应用"→ 前端替换编辑器内容
- 用户点击"拒绝"→ 继续对话

### 4.4 TipTap 可视化编辑器

- A4 尺寸编辑画布（CSS 固定 210mm × 297mm）
- 富文本编辑：加粗、斜体、下划线、字号、颜色、对齐、行距
- 列表：有序/无序
- 图片：上传头像、拖拽调整
- Section 拖拽排序
- 原生 undo/redo（ProseMirror 内置）
- 自动保存（debounce 2 秒）

### 4.5 版本管理

- 每次保存/AI 修改确认后自动创建 HTML 快照
- 版本列表（版本号 + 时间 + 标签）
- 回退：将历史快照写回当前草稿并自动创建新快照
- 版本快照只存 HTML，约 5-10KB

### 4.6 PDF 导出

- 前端发送当前 HTML → 后端创建异步导出任务 → 客户端轮询任务状态 → 完成后下载 PDF
- Chromium 以 A4 尺寸渲染 HTML
- 同一时间只允许一个导出任务（并发控制）
- 导出完成后释放 Chromium 进程
- 商业化：导出权限控制（付费功能）

## 5. 技术架构

详见 [v2 架构设计文档](./superpowers/specs/2026-04-23-architecture-v2-design.md)。

### 5.1 技术栈

| 层 | 技术 |
|---|---|
| 营销站 | Astro |
| 工作台 | Vite + React + TypeScript + Tailwind CSS + shadcn/ui |
| 编辑器 | TipTap（ProseMirror） |
| 后端 | Gin + Go + GORM |
| 数据库 | PostgreSQL >= 15 |
| PDF 导出 | chromedp |
| AI | GLM-5 / GLM-5-Turbo |

### 5.2 部署

```yaml
services:
  nginx:          # ~10MB
  gin:            # ~50MB
  postgres:       # ~500MB
  # 导出时: +chromium ~300MB（临时）
```

2C2G 服务器完全够用。

## 6. 用户主流程

```
1. 创建项目
2. 上传简历文件（或录入补充文本）
3. 系统解析文件，提取文本
4. AI 根据文本生成初始简历 HTML
5. TipTap 加载 HTML，进入工作台
6. 用户选择修改方式：
   a. AI 对话修改：描述需求 → AI 流式返回 HTML → 确认/拒绝
   b. 手动编辑：直接在 TipTap 中编辑
7. 系统自动保存 HTML 快照
8. 用户满意后导出 PDF（chromedp 服务端渲染）
```

## 8. 与 v1 的主要区别

| 维度 | v1 | v2 |
|---|---|---|
| 数据源 | ResumeDraftState JSON + PatchEnvelope | HTML（唯一） |
| 编辑方式 | 结构化表单 + Patch 协议 | TipTap 所见即所得 |
| AI 编辑 | 生成 PatchOp → 应用 → 渲染 | 直接返回 HTML |
| 导出 | LaTeX → xelatex → PDF | chromedp → PDF |
| 部署 | TeX Live 1-5GB | Go 二进制 ~50MB |
| 中间数据结构 | 6 层 | 0 层 |
| 开发复杂度 | 高 | 低 |
